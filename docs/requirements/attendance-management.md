# 勤怠管理アプリ — 要求仕様書

## 概要

社員の出退勤打刻・勤務時間集計・休暇申請・残業申請・月次レポート表示を行う Web アプリケーション。
対象は数十人規模の組織。

## ロール

| ロール | 説明 |
|--------|------|
| 一般社員 | 自身の打刻・申請・レポート閲覧を行う |
| 管理者 | 上記に加え、部下の休暇申請・残業申請の承認、全社員のレポート閲覧を行う |

## ユーザーストーリー

### 認証

- US-1: 社員として、ID/パスワードでログインしたい。自分の勤怠情報にアクセスするため。
- US-2: 社員として、ログアウトしたい。セッションを終了するため。

### 出退勤打刻

- US-3: 社員として、Web ブラウザから出勤ボタンを押して出勤を記録したい。
- US-4: 社員として、Web ブラウザから退勤ボタンを押して退勤を記録したい。
- US-5: 社員として、打刻忘れや誤打刻を自分で修正したい。正確な勤務記録を残すため。

### 勤務時間集計

- US-6: 社員として、日次の勤務時間（実働時間・残業時間）を確認したい。
- US-7: システムとして、所定労働時間（8時間）を超えた分を残業時間として自動計算する。

### 休暇申請

- US-8: 社員として、有給休暇を申請したい。
- US-9: 社員として、欠勤を申請したい。
- US-10: 管理者として、部下の休暇申請を承認または却下したい。
- US-11: 社員として、申請した休暇のステータス（申請中・承認・却下）を確認したい。

### 残業申請

- US-12: 社員として、残業申請を提出したい（事前・事後どちらも可）。
- US-13: 管理者として、部下の残業申請を承認または却下したい。
- US-14: 社員として、申請した残業のステータスを確認したい。

### 月次レポート

- US-15: 社員として、自分の月次勤務レポート（勤務日数・総勤務時間・残業時間・休暇取得日数）を画面で確認したい。
- US-16: 管理者として、全社員の月次勤務レポートを画面で確認したい。

## 勤務ルール

| 項目 | 値 |
|------|---|
| 所定労働時間 | 8時間/日 |
| 残業計算 | 所定労働時間超過分を自動計算 |
| 休日 | 土日祝（日本の祝日） |
| 休暇種別 | 有給、欠勤 |

## 承認フロー

```
申請（社員） → 承認/却下（管理者） → 反映
```

- 休暇申請・残業申請ともに管理者の承認が必要
- 承認されるまでは「申請中」ステータス

## 打刻修正ルール

- 修正できるのは本人のみ
- 管理者は他の社員の打刻を修正できない

## 技術スタック

| レイヤー | 技術 |
|----------|------|
| Backend | Java / Spring Boot |
| Frontend | Next.js (TypeScript) |
| DB | PostgreSQL |
| 認証 | ID/パスワード認証（Spring Security） |

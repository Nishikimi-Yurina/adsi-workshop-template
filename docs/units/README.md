# Unit of Work 分割

## 依存図

```
Phase A（共通基盤）
┌─────────────────────────────────┐
│  Unit 0: 共通基盤               │
│  (プロジェクト骨格・認証・Employee) │
└────────────┬────────────────────┘
             │
Phase B（並列実装可能）
             │
     ┌───────┴───────┐
     ▼               ▼
┌──────────┐   ┌──────────┐
│ Unit 1   │   │ Unit 2   │
│ 打刻     │   │ 申請     │
│ +レポート │   │ +承認    │
└──────────┘   └──────────┘
```

## Phase 一覧

| Phase | Unit | 担当 | 依存 | 内容 |
|-------|------|------|------|------|
| A | Unit 0: 共通基盤 | 2人で協力 | なし | プロジェクト骨格・Flyway・認証・Employee CRUD |
| B | Unit 1: 打刻・レポート | 担当者 1 | Unit 0 | 出退勤打刻・勤務時間計算・月次レポート |
| B | Unit 2: 申請・承認 | 担当者 2 | Unit 0 | 休暇申請・残業申請・承認フロー |

## 並列実装の前提

- Unit 0 完了後、Unit 1 と Unit 2 は **独立して並列実装可能**
- Unit 1 は `attendance_records` テーブルのみ使用
- Unit 2 は `leave_requests` + `overtime_requests` テーブルのみ使用
- 両 Unit とも `employees` テーブル（Unit 0 で作成済み）を参照する

## 統合

Phase B 完了後、ダッシュボード画面で両 Unit の情報を統合表示する。
ダッシュボードの打刻ボタンは Unit 1、直近の申請状況は Unit 2 のコンポーネントを組み合わせる。
